\section{工程实现与代码结构}

本章介绍项目的工程实现情况，包括目录组织、核心模块和训练评估流程。

\subsection{项目目录结构}

项目的代码采用模块化组织，核心目录如下：

\begin{lstlisting}[language=bash,caption={项目目录结构}]
project_root/
  src/
    data/          # 数据加载、预处理、CLIP特征缓存
    models/        # 模型定义(CLIP, A2 encoder, fusion, caption model)
    train/         # 训练脚本(baseline, fusion, alignment等)
    eval/          # 评估(generation, retrieval, sanity check, metrics)
    losses/        # 损失函数(contrastive, EEG augmentation, similarity)
    utils/         # 工具(config loader, seed, checkpoint, logger)
  scripts/         # 启动脚本和数据工具
  configs/         # YAML配置文件
  outputs/         # 训练输出(checkpoint, metrics, report)
  tests/           # 单元测试
\end{lstlisting}

\subsection{训练与评估流程}

\textbf{训练流程}：(1) 数据准备——构建 JSONL manifest，生成 CLIP 特征缓存；(2) A2 EEG encoder 对齐训练（可选）——使用 InfoNCE 对比损失将 EEG 映射到 CLIP 空间；(3) A2 语义融合训练——联合训练 fusion + classification 模块；(4) VTF token 融合训练；(5) 生成式模型训练——Qwen2-VL + LoRA；(6) 全量测试——所有退化条件 $\times$ EEG 模式的组合评估。

\textbf{输出文件}：每次训练在 \texttt{outputs/<run\_name>/} 下生成最佳 checkpoint、训练日志、metrics JSON 和 report. 生成式评估产出 JSONL 文件，每条记录包含 image\_id、mode、reference、prediction 字段。

\subsection{遇到的工程问题}

在实际开发中遇到并解决了以下问题：(1) Qwen2-VL 的 vision adapter 与 CLIP ViT token 接口不完全匹配，需在 VTF 后插入轻量适配器进行格式对齐；(2) BLIP 生成的伪 caption 中约 15\% 存在格式不规范，通过最小长度过滤（$\geq$ 3 词）和正则去噪处理；(3) 全量测试涉及 1997 $\times$ 6 $\times$ 4 $\approx$ 48000 次推理，生成式模型的自回归解码是主要时间瓶颈，完整评估约需 2--4 小时；(4) 所有模型均在单 GPU 上运行，未引入 DeepSpeed 或量化，显存峰值约 13.7 GB，在 48 GB 显卡上留有充足余量。
